## Prerequisites

- Bun 1.3.x or later
- Git
- access to the BigCommerce store credentials used by this storefront
- access to the Stencil access token used for local theme work

You do **not** need a separate widget-builder checkout for this repository.

## Install dependencies

From the repo root:

```bash
bun install
```

## Do not run `stencil init`

This repo no longer relies on the older interactive `stencil init` flow.

Instead, place the repo's local config files directly in the storefront and theme directories:

- repo root `.env`
- `apps/storefronts/rustoleum-home/.env`
- `apps/storefronts/rustoleum-home/stencil-theme/config.stencil.json`
  or `apps/storefronts/rustoleum-home/stencil-theme/.config.stencil.json`
- `apps/storefronts/rustoleum-home/stencil-theme/secrets.stencil.json`

<Warning>
  Switching environments is not a single flag change. Rotate the env and theme config files together so the
  store URL, Stencil token, BigCommerce API credentials, channel, and local secrets all point at the same
  environment.
</Warning>

## Required local files

### 1. Repo root `.env`

Start from `/.env.example`.

This file is used by Bun-driven tooling and publish commands. Typical values include:

- `BC_STORE_HASH`
- `BC_ACCESS_TOKEN`
- `BC_CHANNEL_ID`
- `VITE_MUI_LICENSE_KEY`
- local API or SiteVibes fallback values when needed

### 2. Storefront `.env`

Start from `apps/storefronts/rustoleum-home/.env.example`.

This is the storefront-local fallback env surface used for values such as:

- `VITE_APP_SITEVIBES_API`
- `VITE_APP_SITEVIBES_APP_ID`

### 3. Theme config

Start from `apps/storefronts/rustoleum-home/stencil-theme/config.stencil.example.json`.

Create either:

- `config.stencil.json`, or
- `.config.stencil.json`

The config file carries the theme-local Stencil settings such as:

- `normalStoreUrl`
- `port`
- `packageManager`

### 4. Theme secrets

Start from `apps/storefronts/rustoleum-home/stencil-theme/secrets.stencil.example.json`.

Create:

- `secrets.stencil.json`

This file typically carries theme-local secret values such as:

- `accessToken`
- `sitevibesApiToken`
- `sitevibesAppId`

## Fastest setup path

If you want prompts instead of copying files manually, use the repo tooling initializer:

```bash
bun packages/tooling/cli.ts init
```

That command writes:

- repo root `.env`
- `apps/storefronts/rustoleum-home/stencil-theme/.config.stencil.json`
- `apps/storefronts/rustoleum-home/stencil-theme/secrets.stencil.json`

It is a repo-specific setup helper. It is not the same as `stencil init`.

## Start local development

### Preferred entrypoint

```bash
bun run dev
```

Use this when you are touching anything in:

- `packages/api`
- `packages/ui`
- `packages/widgets`

It watches those packages, syncs current `dist` output, and then starts the theme server.

### Theme-only entrypoint

```bash
bun run stencil:start
```

Use this when package builds are already current and you only need the theme server.

## Other useful commands

```bash
# full workspace build
bun run build

# pull the remote stencil theme
bun run stencil:pull

# run tests
bun run test
```

## Environment rotation checklist

When you move from one store environment to another, update all of these together:

1. repo root `.env`
2. `apps/storefronts/rustoleum-home/.env`
3. `config.stencil.json` or `.config.stencil.json`
4. `secrets.stencil.json`

If those files point at different stores, you can end up publishing widgets to one environment while running the theme against another.
