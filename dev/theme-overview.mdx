## Top-level layout

```text
.
├── packages/
│   ├── api/
│   ├── tooling/
│   ├── ui/
│   └── widgets/
├── apps/
│   └── storefronts/
│       └── rustoleum-home/
│           ├── .env
│           └── stencil-theme/
└── mintlify-docs/
    ├── content/
    └── dev/
```

## Package boundaries

### `packages/api`

Put shared API utilities and API-oriented code here.

Use this package when the behavior is not theme-specific and should be shared by widgets, wrappers, or other React surfaces.

### `packages/ui`

Put reusable React UI here:

- shared components
- shared composites
- icon exports
- theme tokens
- typography helpers

Use this package before creating one-off widget-local primitives.

### `packages/widgets`

This is the source of truth for package-backed Page Builder widgets.

Each widget folder lives under:

```text
packages/widgets/src/widgets/<widget-name>/
```

Typical contents:

- `widget.ts`
- `template.html`
- widget-local `.tsx` and `.styles.tsx` files
- optional widget-local `README.md`

### `packages/tooling`

This package owns the repo CLI for:

- init
- widget create
- widget publish
- widget validate
- widget delete
- invalid placement recovery

If the task is about BigCommerce Widgets API operations or placement cleanup, it belongs here.

## Storefront surfaces

### `apps/storefronts/rustoleum-home`

This folder holds storefront-local env and the active Stencil theme package.

### `apps/storefronts/rustoleum-home/stencil-theme`

This is where theme-specific work belongs:

- Handlebars templates
- theme-local wrappers
- widget registry wiring
- SCSS
- page-manager code
- bundle, validate, and push scripts

## What belongs where

| Task | Put it here |
| --- | --- |
| shared button, input, dialog, token, or icon work | `packages/ui` |
| shared widget implementation and Page Builder schema | `packages/widgets` |
| BigCommerce widget publish or placement logic | `packages/tooling` |
| widget registry wiring | `apps/storefronts/rustoleum-home/stencil-theme/assets/js/framework/widgets/registry.js` |
| storefront-specific data shaping or runtime wrappers | `apps/storefronts/rustoleum-home/stencil-theme/assets/js/components/widget-wrappers` |
| Handlebars page templates and theme layout changes | `apps/storefronts/rustoleum-home/stencil-theme/templates` |
| developer docs | `mintlify-docs/dev` |
| content-team docs | `mintlify-docs/content` |

## Widget-specific placement rules

For a new or updated widget, expect to touch these surfaces together:

1. `packages/widgets/src/widgets/<widget-name>/...`
2. `packages/widgets/src/index.ts`
3. `packages/widgets/package.json`
4. `apps/storefronts/rustoleum-home/stencil-theme/assets/js/framework/widgets/registry.js`
5. optional theme wrapper files when the widget needs storefront-specific data

## Theme runtime structure

| Path | Role |
| --- | --- |
| `assets/js/framework/widgets/registry.js` | widget type to runtime loader map |
| `assets/js/framework/widgets/runtime.js` | widget mount runtime |
| `assets/js/components/widget-wrappers` | storefront-aware wrapper components |
| `templates/` | Handlebars and Stencil templates |
| `scripts/start-theme.mjs` | local theme start orchestration |
| `scripts/bundle-theme.mjs` | bundle generation |
| `scripts/validate-theme-bundle.mjs` | bundle safety checks |
| `scripts/push-theme-bundle.mjs` | push and activation flow |

## Rules that keep the repo understandable

- Put shared behavior in packages first.
- Keep `template.html` focused on the Stencil-to-React handoff, not full visual logic.
- Use theme wrappers only when the widget needs storefront runtime data or theme-local orchestration.
- Keep developer docs in `mintlify-docs/dev` self-contained instead of relying on local-only repo files.
