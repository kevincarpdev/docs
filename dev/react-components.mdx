## PDP components

These components render the product detail page. They are mounted by `app.js` into `#react-product-view`.

### ProductView

**File:** `assets/js/components/ProductView.jsx`

The top-level PDP component. Manages:
- Image gallery state (active index, variant images, lightbox)
- Breadcrumb building from multiple data sources
- Category detection (accessory vs. non-accessory)
- Layout grid (image section + details section)

**Props:**
| Prop | Type | Source |
|------|------|--------|
| `product` | Object | `data-product` JSON from Handlebars |
| `images` | Array | `data-images` JSON from Handlebars |
| `breadcrumbs` | Array | `data-breadcrumbs` JSON from Handlebars |
| `categories` | Array | `data-categories` JSON from Handlebars |
| `showDeals` | Boolean | Default `true` |

### ProductImageGallery

**File:** `assets/js/components/ProductImageGallery.jsx`

Desktop and mobile image gallery with:
- Thumbnail strip (max 5 visible, "+N more" overlay)
- Swipeable mobile carousel
- Lightbox integration
- Active image tracking from parent

### ProductOptions

**File:** `assets/js/components/ProductOptions.jsx`

Renders product options (color swatches, dropdowns, quantity). Listens to the hidden Stencil form for option change events and syncs React state.

### ProductAccordions

**File:** `assets/js/components/ProductAccordions.jsx`

Expandable sections below the product options: Description, Specifications, Kit Includes, etc. Content comes from product metafields.

### DealsSection

**File:** `assets/js/components/DealsSection.jsx`

Shows supplemental info below options: additional supplies image, satisfaction message, shipping message. Data comes from channel metafields.

### StickyPDPHeader

**File:** `assets/js/components/StickyPDPHeader.jsx`

Fixed header that appears when scrolling past the Add to Cart button. Shows product name, price, and a sticky Add to Cart CTA.

## Accessory vs. non-accessory logic

`ProductView` detects whether a product is in an "Accessories" category:

```js
const isAccessoryCategory = useMemo(() => {
  const accessorySlug = 'accessor';
  const check = (str) => (str || '').toLowerCase().includes(accessorySlug);
  // checks categories, fetchedCategories, and product.category
  // ...
}, [...]);
```

When `isAccessoryCategory` is `true`:
- The Room Visualizer callout is hidden
- The mobile visualizer is hidden

## Widget components

These are hydrated by `widgets-loader.js` from Page Builder widget DOM nodes.

### HeroCarousel / HeroSectionCarousel

Full-width hero banner with slides. Supports badge, headline, subheadline, background image, CTA buttons, and autoplay.

### BenefitsBlock / Benefits2Block

Product benefit icons with labels. Two layout variants for different page contexts.

### HowItWorks / HowItWorks2

Step-by-step process section with category tabs (Floor, Countertop, Cabinet). On PDP pages, automatically selects the tab matching the product's category.

### GalleryBlock

Image gallery with filtering (Surface/Room tabs, category filters). Used on the Inspiration Gallery page.

### BlogArticles / RelatedArticles

Article card grids. BlogArticles for index pages, RelatedArticles for article detail pages.

### ProductCategories / CategoryBlock

Category navigation cards with images.

### FaqBlock / FaqHighlightBlock / FAQPageContentBlock

Various FAQ display formats — accordion, highlighted, and full-page layouts.

### ProcessSectionBlock

Multi-step process visualization with category-aware content (Floor, Countertop, Cabinet).

### PDFInstructionsBlock

Downloadable PDF links for product instructions. Commonly used on flooring kit PDPs.

## Component conventions

All React components follow these patterns:

- Wrapped in `React.memo()` for performance
- Have a `displayName` set for debugging
- Use `styled()` from MUI (no `sx` props or inline styles)
- Import colors/spacing from `rustoleumHomeTheme.js`
- Use `@phosphor-icons/react` for icons
- Components exceeding ~200 lines are split into smaller focused components
