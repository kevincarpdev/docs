## Two styling layers

The storefront uses two main styling surfaces:

| Surface | Use for |
| --- | --- |
| `packages/ui` theme + MUI `styled()` | shared React UI and package widgets |
| theme-local SCSS in `stencil-theme/assets/scss` | Stencil templates, legacy surfaces, and theme-level layout styling |

## Shared theme source

The shared theme is assembled in:

- `packages/ui/src/theme/theme.ts`
- `packages/ui/src/theme/tokens.ts`

These files define:

- semantic colors
- typography presets and scales
- spacing rhythm
- radius
- viewport behavior

## Styling rules for React work

- Use `styled()` and theme tokens for durable React styling.
- Prefer existing shared components from `packages/ui` before building new widget-local primitives.
- Use semantic color and typography tokens before introducing custom constants.
- Keep responsive behavior aligned to the repo viewport helpers instead of ad hoc breakpoints.

## Styling rules for theme work

Use theme SCSS when the surface is primarily Stencil-template-driven:

- global layout adjustments
- theme shell styles
- legacy template styling
- non-React storefront surfaces

Do not create a second custom theme object for React components. Use the shared `packages/ui` theme tokens instead.

## Common files

| Path | Role |
| --- | --- |
| `packages/ui/src/theme/theme.ts` | MUI theme assembly |
| `packages/ui/src/theme/tokens.ts` | token definitions |
| `apps/storefronts/rustoleum-home/stencil-theme/assets/scss` | theme-local SCSS |
| `packages/widgets/src/widgets/*/*.styles.tsx` | widget-local styled components |

## Good defaults

- color from `semanticColors`
- typography from `typographyPresets` and scale helpers
- radius from `radius`
- icon usage from `@thesparklaboratory/rusto-ui/Icon`
- layout spacing from token-aware theme values
