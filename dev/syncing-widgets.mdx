## Command entrypoints

You can run the CLI directly:

```bash
bun packages/tooling/cli.ts <command>
```

Or use script aliases where they exist:

```bash
bun run widgets:publish
bun run stencil:push
```

## Top-level commands

| Command | Purpose |
| --- | --- |
| `init` | write local env and theme config files |
| `widget create` | scaffold a new widget folder |
| `widget publish` | publish widgets with optional build and placement handling |
| `widget validate` | validate widget schema, translation coverage, and parity |
| `widget delete` | delete published widgets or widget templates |
| `widget resolve-invalid-placements` | clean up stale widget/template/placement relationships |
| `placement delete` | remove placements from a page |
| `placement fix-invalid-array-placements` | normalize invalid array-shaped placement config payloads |

## `init`

```bash
bun packages/tooling/cli.ts init
```

Writes:

- repo root `.env`
- `apps/storefronts/rustoleum-home/stencil-theme/.config.stencil.json`
- `apps/storefronts/rustoleum-home/stencil-theme/secrets.stencil.json`

## `widget create`

```bash
bun packages/tooling/cli.ts widget create <widget-name> [--debug]
```

Arguments:

- `<widget-name>`: kebab-case folder name

Options:

- `--debug`

## `widget publish`

```bash
bun packages/tooling/cli.ts widget publish [options]
```

Options:

| Option | Meaning |
| --- | --- |
| `--build` | force `bun run build` before publish |
| `--channel-id <id>` | override publish channel |
| `--debug` | print raw trace details |
| `--dry-run` | validate inputs and show intended actions only |
| `--entity-id <id>` | placement entity target |
| `--list-only` | show inventory only |
| `--non-interactive` | disable prompts |
| `--region <name>` | placement region override |
| `--registry-file <path>` | explicit registry override |
| `--skip` | skip placement handling |
| `--sort-order <n>` | placement sort order override |
| `--stencil-publish` | run `bun run stencil:push` after publish |
| `--storefront <slug>` | storefront slug under `apps/storefronts` |
| `--template-file <path>` | placement template override |
| `--template-version-mode` | template update mode: `create-new-version` or `in-place` |
| `--update-placements` | opt in to placement updates |
| `--upgrade-template` | upgrade widgets to latest template version |
| `--widgets <comma-separated>` | publish subset |
| `-y, --yes` | skip confirmation prompt |

## `widget validate`

```bash
bun packages/tooling/cli.ts widget validate [widget-name] [options]
```

Options:

- `--bypass-translation-gate`
- `--debug`

## `widget delete`

```bash
bun packages/tooling/cli.ts widget delete [options]
```

Options:

| Option | Meaning |
| --- | --- |
| `--channel-id <id>` | channel override |
| `--debug` | raw delete trace |
| `--non-interactive` | disable prompts |
| `--template` | delete widget templates grouped by widget name |
| `--widgets <names-or-uuids>` | explicit selection |
| `-y, --yes` | skip confirmation |

## `widget resolve-invalid-placements`

```bash
bun packages/tooling/cli.ts widget resolve-invalid-placements [options]
```

Options:

- `--channel-id <id>`
- `--debug`

Use this when published widget/template relationships are stale and Page Builder data needs recovery.

## `placement delete`

```bash
bun packages/tooling/cli.ts placement delete [options]
```

Options:

| Option | Meaning |
| --- | --- |
| `--all` | delete every placement on the selected page |
| `--channel-id <id>` | channel override |
| `--debug` | raw trace |
| `--non-interactive` | disable prompts |
| `--page <template>` | page template, such as `pages/home` |
| `--widgets <comma-separated>` | delete placements only for selected widgets |
| `-y, --yes` | skip confirmation |

## `placement fix-invalid-array-placements`

```bash
bun packages/tooling/cli.ts placement fix-invalid-array-placements [options]
```

Options:

- `--channel-id <id>`
- `--debug`
- `--widget <name-or-uuid>`

## Practical recipes

### List publishable widgets only

```bash
bun packages/tooling/cli.ts widget publish --list-only --storefront rustoleum-home
```

### Publish a subset in CI-safe mode

```bash
CI=true bun packages/tooling/cli.ts widget publish \
  --storefront rustoleum-home \
  --widgets categories,gallery \
  --non-interactive \
  --yes
```

### Delete all placements on a page

```bash
CI=true bun packages/tooling/cli.ts placement delete \
  --page pages/home \
  --all \
  --non-interactive \
  --yes
```

### Delete only one widget's placements on a page

```bash
CI=true bun packages/tooling/cli.ts placement delete \
  --page pages/home \
  --widgets categories \
  --non-interactive \
  --yes
```
