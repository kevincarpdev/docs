## The Page Builder interface

The Page Builder screen has these key areas:

| Area | Location | Purpose |
|------|----------|---------|
| **Page selector** | Top left dropdown | Switch between pages (Homepage, Gallery, etc.) |
| **Design / Preview toggle** | Top center | Switch between editing mode and preview mode |
| **Device preview** | Top center (icons) | Preview desktop, tablet, and mobile layouts |
| **Settings panel** | Left sidebar | Edit the selected widget's content and settings |
| **Canvas** | Center | Visual preview of the page with draggable widgets |
| **Discard / Save / Publish** | Top right | Manage your changes |

## Selecting a page

Use the **Page** dropdown at the top left to switch between pages. Available pages include:

- Homepage
- Gallery
- Blog Index
- FAQ pages
- Rewards pages
- Category-specific pages

## Design vs. Preview mode

- **Design mode** — Widgets are clickable and editable. You see blue outlines around widget regions and "Drop widgets here" zones.
- **Preview mode** — Shows the page exactly as customers will see it. No editing controls visible.

Toggle between them using the **Design** / **Preview** buttons at the top center.

## Placing widgets

### Adding a new widget

1. In Design mode, find a **"Drop widgets here"** zone on the page
2. Click the zone or drag a widget from the widget catalog
3. The widget appears on the page and its settings open in the left panel

### Moving a widget

Drag a widget by its handle to reposition it within or between drop zones.

### Removing a widget

Hover over a widget and click the trash icon to remove it from the page.

## Widget categories

Custom Rust-Oleum HOME widgets appear under the **Rustoleum Home** category in the widget catalog. These include all the custom blocks like Hero Carousel, Benefits, How It Works, Gallery, FAQ blocks, etc.

BigCommerce also provides built-in widgets (Text, Image, HTML, etc.) that can be used alongside custom widgets.

## Saving and publishing

| Action | What it does |
|--------|-------------|
| **Save** | Saves your changes as a draft. Not visible to customers. |
| **Discard** | Reverts all unsaved changes. |
| **Publish** | Makes your saved changes live on the storefront. |

<Warning>
  Clicking **Save** does not make changes live. You must click **Publish** for customers to see your updates.
</Warning>

## Tips

- Use **Preview mode** frequently to check how your changes look
- Test on all device sizes using the device preview icons
- Save often to avoid losing work
- If something looks wrong after publishing, you can make corrections and publish again immediately
