This reference covers every custom widget available in the Rust-Oleum HOME Page Builder. Each widget is a configurable building block — drag it onto a page, fill in its fields, and publish. No code required.

<Note>
  All widgets are accessed from the **Page Builder** panel in the BigCommerce admin. Navigate to **Storefront → Page Builder**, open a page, and drag widgets from the left panel onto the canvas.
</Note>

## Common Fields Across All Widgets

### Anchor

Every widget that supports deep linking has an **Anchor** field in its **Settings** tab. When filled in, it adds a page anchor (`id` attribute) to the widget's outermost element.

- **Format:** Use lowercase slugs — `paint-colors`, `how-it-works`, `shop-now`
- **Usage:** Link to a specific section using `#anchor` in a button URL elsewhere on the page
- **Example:** Set anchor to `get-started`, then link a hero CTA to `#get-started`

### Product IDs

Any field labelled **Product**, **Product Selector**, or **Override Product** expects a **BigCommerce product ID** (a number, not a product name or SKU).

To find a product ID:
1. Go to **Products** in the BigCommerce admin
2. Open the product
3. Check the URL — it ends in `/products/{ID}/edit`

### Rich Text Fields

Fields labelled as `text` type (vs. `input`) support basic rich text: **bold**, *italic*, links, and line breaks. Use sparingly — over-formatting diminishes visual hierarchy.

## Image Best Practices

| Use Case | Recommended Size | Format |
|----------|-----------------|--------|
| Hero / full-width background | 1440 × 700 px | JPG, WebP |
| Standard section image | 800 × 600 px | JPG, WebP |
| Square product / gallery | 800 × 800 px | JPG, WebP |
| Avatar / portrait | 200 × 200 px | JPG, PNG |
| Icon | 64 × 64 px | SVG, PNG |

- Always compress images before uploading — target under 300 KB for photos.
- Use WebP format when possible for better performance.
- Name image files descriptively before uploading: `kitchen-painted-walls-sage-green.jpg`, not `IMG_4823.jpg`.
